# 设计原则

## 为什么需要这个 Skill？

当前 AI Agent 在处理复杂任务时普遍存在三个问题：

### 1. 上下文溢出
一次对话中包含过多信息，导致早期的需求分析被遗忘或混淆。

### 2. 方向偏差
Agent 按照自己的理解直接开始执行，做到一半发现不是用户想要的。

### 3. 信息不对称
用户知道最终目标，但不知道需要经历哪些步骤。Agent 知道步骤却不按步骤引导。

## 解决方案拆解

### 角色分工

```
用户 → 提供需求（知道要什么）
Agent → 提供流程（知道怎么做）

Agent 是流程专家，用户是需求方
```

### 核心机制

1. **自动检测**：不依赖用户手动切换，agent 根据任务复杂度自行判断
2. **分步引导**：每次只问 1 个问题，降低用户认知负担
3. **自动退出**：不追问"可以开始了吗"，信息够了直接进执行

### 和传统做法的区别

| 维度 | 传统做法 | Plan Mode |
|------|---------|-----------|
| 触发方式 | 用户手动命令 | Agent 自动检测 |
| 需求收集 | 一次问完所有问题 | 每次 1 步 |
| 退出方式 | 用户说"开始做" | Agent 自行判断 |
| 领域适配 | 通用模板 | 按领域自动切换脚手架 |

## 适用场景

- 网站/小程序/App 开发规划
- 深度行业研究报告
- 数据分析项目
- 产品设计方案
- 任何需要 >3 个步骤的复杂任务
