# 示例：二手书交易小程序

## 用户输入

> "帮我做一个二手书交易小程序"

## Agent 执行过程

### 阶段 1：进入 Plan Mode

Agent 自动检测到这是一个复杂任务（涉及用户系统、商品展示、交易流程），
自动进入规划模式并创建 workspace。

### 阶段 2：需求收集（4 轮对话）

**Round 1**
Agent：这个二手书小程序主要是给谁用的？大学校园内的学生，还是有更广的用户群？
用户：主要是广西大学的学生用
→ 写入 `context/01-需求对齐.md`

**Round 2**
Agent：最核心的 3 个功能是什么？比如浏览/搜索、发布闲置书、下单交易？
用户：发布和浏览最重要，交易可以先放一放
→ 写入 `context/02-功能确认.md`

**Round 3**
Agent：需要用户注册登录吗？还是直接用微信授权？
用户：微信授权就行
→ 写入 `context/03-技术方案.md`

**Round 4：Agent 自动评估**
- ☑ 目标用户明确
- ☑ 核心功能清晰
- ☑ 登录方案确定
- ☑ → 信息充分，自动退出

### 阶段 3：退出 Plan Mode

```
✅ 信息收集完毕，退出规划模式。

任务总览：广西大学二手书交易小程序
核心需求：微信授权登录 → 发布闲置书 → 浏览/搜索图书
领域脚手架已整理至：~/Desktop/D老师记忆/二手书小程序/plan.md

进入执行阶段。
```

### 阶段 4：执行

按照 `plan.md` 的阶段表顺序交付：
1. 技术架构设计
2. 页面原型
3. 数据库设计
4. 核心功能实现
5. 部署上线
