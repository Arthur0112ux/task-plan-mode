# task-plan-mode — 任务规划模式 v2.1 🎯

> **模仿 Codex Plan Mode：收到任务 → 直接出计划 + 开始第一步。**
>
> 不需要前置审批。AI 自己评估、自己分段、边做边汇报。计划跑在行动里，不是前置条件。

---

## 一、核心流程

```
用户提出任务
      ↓
AI 直接输出：阶段规划 + 第一个需求问题
      ↓
逐轮对话推进（每次只推进 1 个环节）
      ↓
每个阶段完成后标注进度
      ↓
所有阶段完成 → 汇总交付
```

**关键原则：** 无前置审批。计划作为回复的一部分自然呈现。

---

## 二、触发规则

### 自动触发

| 触发条件 | 示例 |
|---------|------|
| 任务涉及 >2 个步骤 | 做网站、写报告、开发项目 |
| 模糊需求、缺关键信息 | "帮我做一个关于新能源的东西" |
| 包含做/开发/设计/写 + 复杂对象 | "做一个二手书交易小程序" |

### 手动触发

`规划模式` / `Plan Mode` / `帮我拆解` / `先规划一下`

---

## 三、收到任务后的第一回复（核心）

**不要停下来问"方案可以吗？"——直接出计划+开始第一步。**

### 模板

```
收到。这个任务我打算分 3 步走：
① 需求确认 — 先摸清目标读者和核心内容
② 数据搜集 — 找近一年行业数据
③ 撰写交付 — 按章节产出

先开始第一步：这份报告的目标读者是谁？
```

### 要点
- 计划 + 第一个问题一起输出，不单独等待确认
- 用户不认同计划会自然打断
- 不打断就默认按此推进

---

## 四、阶段划分规则

### 默认划分

| 任务类型 | 默认阶段 |
|---------|---------|
| 网站/应用 | 需求确认 → 前端 → 后端 → 联调交付 |
| 文章/报告 | 需求确认 → 大纲 → 正文 → 排版交付 |
| 研究分析 | 需求确认 → 数据搜集 → 分析 → 报告 |
| 设计类 | 需求确认 → 概念设计 → 细化 → 交付 |
| 其他 | AI 动态生成 |

---

## 五、逐轮推进

### 需求收集

每轮 1 个问题，信息足够即进入下一阶段：

| 类型 | 收集顺序 |
|------|---------|
| 网站 | 目标用户 → 核心功能 → 技术栈 → 页面结构 → 设计风格 |
| 文章 | 读者与观点 → 篇幅风格 → 数据来源 |
| 研究 | 问题与范围 → 数据类型 → 分析方法 |

每轮最多追问 2 次，用户给不出答案时默认建议。

### 阶段汇报

每阶段完成后输出：

```
✅ [阶段名] 完成 — {摘要}
➡ 下一步：[下一阶段]
📌 当前：Phase N/M
```

## 七、完整示例

```
用户：帮我做一份新能源行业趋势分析报告

AI：
收到。我打算分 3 步：
① 需求确认 — 明确读者和范围
② 数据搜集 — 找行业数据和技术趋势
③ 撰写交付 — 按章节产出

先开始第一步：这份报告的目标读者是谁？公众号读者还是行业内部？

用户：公众号读者，3000-5000字，客观分析

AI：核心要突出什么方向？固态电池和氢能？

...

AI（完成后）：
✅ 需求确认完成 — 公众号长文，聚焦固态电池与氢能商业化
➡ 下一步：数据搜集
📌 当前：Phase 2/3
```
