# task-plan-mode — 任务规划模式 v3.0

> **任务提示模式：收到任务 → AI 按类型匹配执行方案 → 出计划 + 开始第一步。**

---

## 一、核心流程

```
用户提出任务
      ↓
AI 识别任务类型 → 匹配对照表中的阶段+工具+输出格式
      ↓
输出阶段规划 + 第一个需求问题
      ↓
逐轮对话推进（每次只推进 1 个环节）
      ↓
每个阶段完成后标注进度
      ↓
所有阶段完成 → 汇总交付
```

---

## 二、触发规则

### 自动触发

| 触发条件 | 示例 |
|---------|------|
| 任务涉及 >2 个步骤 | 做网站、写报告、开发项目 |
| 模糊需求、缺关键信息 | "帮我做一个关于新能源的东西" |
| 包含做/开发/设计/写 + 复杂对象 | "做一个二手书交易小程序" |

### 手动触发

`规划模式` / `Plan Mode` / `帮我拆解` / `先规划一下`

## 四、各阶段通用工具指引

| 工具 | 适用场景 | 用法 |
|------|---------|------|
| `web_fetch(url)` | 搜资料、抓网页、找数据 | 所有"资料搜集/数据搜集"阶段必用 |
| `exec(command)` | 执行命令、调 API、运行代码 | 搜索(`curl`)、文生图(`curl`)、代码执行 |
| `lark-doc` | 写文章、报告、存档 | 所有"写作/排版/交付"阶段必用 |
| `lark-im` | 发消息、搜聊天记录 | 推送、查历史 |
| `write(path, content)` | 本地文件产出 | 代码文件、本地存档 |
| `read(path)` | 读技能、读记忆、读已有文档 | 任何阶段开始前先读 skill |

## 五、逐轮推进规则

### 需求收集

每轮 1 个问题，信息足够即进入下一阶段。

每轮最多追问 2 次，用户给不出答案时默认建议 + 标注"用户未确认"。

### 阶段汇报

每阶段完成后必须输出：

```
✅ [阶段名] 完成
产出：{文件/文档名称}
工具：{用了什么工具}
-> 下一步：[下一阶段]
📌 当前：Phase N/M
```

---

## 六、完成标准

### 一个阶段"完成"的条件
- 该阶段的**产出格式**已生成
- **完成标准**已达成
- 如果是需求阶段：用户说"够了我确认了"或 agent 自评 checklist 全过

### 整个任务"完成"的条件
- 所有阶段都标记完成
- 最终产出可打开/可运行/可直接使用
- 已通知 Arthur 交付

```
用户：帮我写一篇关于固态电池商业化进展的公众号文章

先开始第一步：这篇文章预期面向什么读者？公众号粉丝还是行业人士？字数大概多少？
```
